# [SYSTEM COMPONENT: Agent -> DBlayer Mutation Payload]
**Last Updated:** May 2026 | **Architecture Version:** v5.0
**Status:** APPROVED

## 1. Context & Scope
This document defines the exact JSON payload structure sent from the Agent to the DBlayer server during Step 4 of the Turn Execution Loop. It contains the explicit database commands generated by the Agent after evaluating player inputs against the Cold State, ensuring all state changes, logging, and history compressions are written to the database in a single transaction.

## 2. Core Functional Specification
*   **Strict Schema Validation:** The DBlayer will validate this payload against predefined schemas. If it attempts to modify a field outside its bounds (e.g., dropping health below 0 without a death state), the DBlayer handles the sanitization or rejects it.
*   **Three-Part Payload:** The payload must always contain:
    1.  **Mutations:** Explicit instructions on how to alter the Cold State (deltas for integers, additions/removals for arrays).
    2.  **Turn Log:** The raw historical data to append to the Hot Context ledger.
    3.  **World Bible Update:** A nullable string. If the turn triggers a compression (e.g., Turn 10, 20), this field contains the new summary to append to the Warm State. Otherwise, it is `null`.
